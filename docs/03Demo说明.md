# 03 Demo说明

## 一、Demo 目标

FundWise AI Demo 用移动端 Web 形式模拟一个基金投资辅助 App。Demo 需要在面试中完成 5-8 分钟演示，清晰展示从用户痛点到 AI 功能优化的产品闭环。

演示重点：

- 新手如何用自然语言完成选基。
- 推荐结果如何解释适配原因、风险和持有周期。
- 买入后如何看懂收益来源、组合健康度和风险预警。
- 投教内容如何基于持仓和操作场景推荐，而不是通用推送。

## 二、页面结构

### 1. App 外壳

建议组件：

- `MobileShell`：模拟 iPhone 页面宽度、状态栏和底部导航。
- `BottomTabBar`：底部导航，包含首页、选基、持仓、投教。
- `TopBar`：页面标题、返回、辅助入口。

作用：统一移动端 App 风格，让 Demo 更像完整产品而不是单页拼贴。

### 2. 首页 Dashboard

页面目标：

- 给出 FundWise AI 的产品入口。
- 展示三个核心模块：AI智能选基、AI持仓管家、新手投教成长。
- 显示一条今日 AI 提醒，例如「你的科技主题基金波动扩大，建议查看风险预警」。

主要点击：

- 点击「AI智能选基」进入选基流程。
- 点击「查看持仓日报」进入持仓管家。
- 点击「今日学习任务」进入投教成长。

### 3. AI智能选基页

页面目标：

- 展示新手模式和专业模式切换。
- 在新手模式中突出自然语言输入和快捷需求标签。
- 在专业模式中保留专业筛选项。

主要点击：

- 切换「新手模式 / 专业模式」。
- 点击需求标签，例如「低波动」「适合定投」「长期持有」。
- 点击「生成筛选条件」展示 AI 转译结果。
- 点击「查看推荐」进入结果页。

### 4. 选基结果页

页面目标：

- 展示推荐基金列表。
- 每张基金卡片展示核心指标、适配原因、核心风险和建议持有周期。

主要点击：

- 点击基金卡片进入详情。
- 点击「加入自选」切换状态。
- 点击「对比」加入对比栏。
- 点击「查看定投建议」展示承接动作。

### 5. 基金详情页

页面目标：

- 解释基金为什么适合当前需求。
- 用新手可理解语言解释关键指标。
- 展示不适合人群和风险提示。

主要点击：

- 展开「指标解释」。
- 切换「适配原因 / 风险 / 持有建议」。
- 返回结果页。

### 6. AI持仓管家页

页面目标：

- 展示用户当前持仓概览。
- 用 AI 日报解释今日收益和组合健康度。

主要点击：

- 点击「收益拆解」查看贡献和拖累项。
- 点击「持仓健康度」查看组合问题。
- 点击「风险预警中心」进入风险预警页。

### 7. 风险预警页

页面目标：

- 按高 / 中 / 低风险分层展示预警。
- 给出可执行建议和相关投教内容入口。

主要点击：

- 切换风险等级。
- 展开单条风险说明。
- 点击「查看应对建议」。
- 点击「学习相关知识」跳转投教。

### 8. 新手投教成长页

页面目标：

- 展示与当前持仓和风险场景关联的学习内容。
- 展示任务、积分和进度反馈。

主要点击：

- 点击推荐课程。
- 完成一个互动问答。
- 领取积分。
- 查看学习进度变化。

## 三、面试演示顺序

### 第 1 步：说明项目背景

讲法建议：

「我之前分析基金类 App 时发现，用户不是没有理财需求，而是在选基、理解结果、买后陪伴和投教转化上存在断点。所以这个 Demo 用 AI 做三件事：降低选基门槛、解释持仓变化、把投教嵌入真实场景。」

### 第 2 步：演示 AI智能选基

操作路径：

1. 首页点击「AI智能选基」。
2. 选择新手模式。
3. 输入自然语言需求。
4. 展示 AI 转译条件。
5. 查看推荐结果。
6. 打开基金详情，讲适配原因、风险和持有周期。

要表达的产品价值：

- 从「用户懂指标才能筛」变成「用户说目标，系统帮他转译」。
- 结果不是只给排名，而是解释为什么适合和风险在哪里。

### 第 3 步：演示专业模式

操作路径：

1. 返回 AI智能选基。
2. 切换专业模式。
3. 调整收益、回撤、风险等级等条件。
4. 查看结果解释。

要表达的产品价值：

- 新手和进阶用户都能使用。
- AI 不替代专业筛选，而是提升配置效率和解释质量。

### 第 4 步：演示 AI持仓管家

操作路径：

1. 底部导航进入「持仓」。
2. 查看 AI 持仓日报。
3. 展开收益拆解。
4. 查看持仓健康度。
5. 进入风险预警中心。

要表达的产品价值：

- 买后不只展示收益曲线，而是解释收益来源和风险变化。
- 用户能获得下一步操作建议，减少短期波动带来的盲目操作。

### 第 5 步：演示投教成长

操作路径：

1. 从风险预警点击「学习相关知识」。
2. 进入与当前风险相关的课程。
3. 完成互动问答。
4. 查看积分和进度反馈。

要表达的产品价值：

- 投教不再是通用内容，而是和用户当前持仓、风险、操作场景绑定。
- 任务和进度机制提升持续参与感。

## 四、关键演示数据

建议使用同一虚拟用户贯穿：

- 用户：林然
- 风险承受：中等偏低
- 当前目标：长期定投、降低波动、学习基础基金知识
- 当前持仓：偏成长风格，科技主题基金占比略高
- 当前风险：行业集中度偏高，近期回撤扩大

这样可以形成连续故事：

1. 林然先用新手模式找低波动定投基金。
2. 系统推荐更适合长期配置的基金。
3. 林然已有持仓出现波动，持仓管家解释原因。
4. 风险预警提示行业集中度。
5. 投教推荐「如何理解回撤和分散配置」。

## 五、验收标准

Demo 开发完成后，应满足：

- 三个核心模块均可从首页进入。
- 新手模式和专业模式均可切换。
- 选基结果至少有 3 个可点击基金卡片。
- 持仓页包含收益拆解、健康度和风险预警入口。
- 风险预警至少包含高 / 中 / 低三类状态。
- 投教页包含课程、任务、积分、进度反馈。
- 主要流程在移动端宽度下无明显遮挡、错位或文本溢出。
- 页面中不出现让人误以为官方产品的表达。
