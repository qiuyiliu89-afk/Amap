# 02 产品方案

## 一、产品定位

FundWise AI 是一个面向基金投资新手和进阶用户的智能投资辅助 Demo。它不替用户做真实投资决策，而是帮助用户更高效地表达需求、理解基金结果、看懂持仓变化，并在真实场景中完成投教成长。

核心定位：

- 选前：降低筛选和理解门槛。
- 买后：强化收益解释、风险感知和操作陪伴。
- 成长：把投教内容嵌入持仓和操作场景。

## 二、目标用户

### 新手用户

特征：

- 有理财需求，但对基金指标理解有限。
- 更习惯用生活化语言描述需求。
- 容易被收益排名吸引，也容易被短期波动影响。

核心需求：

- 帮我把需求翻译成筛选条件。
- 告诉我为什么适合、风险在哪里。
- 买了以后每天帮我解释发生了什么。

### 进阶用户

特征：

- 已经了解部分基金指标。
- 希望保留专业筛选效率。
- 需要更好的比较、风险提示和持仓管理。

核心需求：

- 保留专业条件筛选。
- 提升复杂条件配置效率。
- 对结果和持仓给出更结构化解释。

## 三、功能模块

### 模块 1：AI智能选基

目标：重构传统条件选基流程，降低新手使用门槛，同时兼顾专业用户的精细筛选。

核心能力：

- 新手模式：自然语言输入、需求标签、AI 条件转译。
- 专业模式：基金类型、收益、回撤、风险等级、基金经理年限等条件筛选。
- 结果页：推荐基金列表、适配原因、核心风险、建议持有周期、操作承接。
- 详情页：基金概览、指标解释、风险说明、同类对比。

关键设计：

- 将「我想稳一点」转译为低波动、较低回撤、中低风险等级等条件。
- 将「适合长期定投」转译为成立时间、基金经理稳定性、长期收益表现、波动承受范围等条件。
- 每个结果必须解释「为什么推荐」和「不适合谁」。

### 模块 2：AI持仓管家

目标：解决用户买入后只看到盈亏、看不懂变化原因的问题。

核心能力：

- 今日收益拆解：按基金、行业、风格或仓位解释盈亏来源。
- 持仓健康度：用分数、标签和短解释描述组合状态。
- 风险预警中心：按高 / 中 / 低风险分层展示预警。
- 操作建议：继续持有、观察、分批调仓、降低集中度、学习相关知识。

关键设计：

- 不只展示「亏了多少」，还解释「主要由哪类基金拖累」。
- 不只提示「风险较高」，还说明「集中在单一行业」「近期回撤扩大」「波动高于用户测评承受范围」。
- 建议以辅助判断为主，避免承诺收益。

### 模块 3：新手投教成长体系

目标：让投教内容从通用文章列表变成贴合用户当前场景的成长路径。

核心能力：

- 我的专属投教：基于持仓、操作、风险测评推荐内容。
- 场景学习：在定投、持仓波动、风险预警、调仓建议等节点嵌入学习。
- 任务机制：完成课程、答题、复盘获得积分。
- 进度反馈：等级、成长值、错题本、阶段目标。

关键设计：

- 用户看到风险预警时，推荐「如何理解最大回撤」。
- 用户准备定投时，推荐「定投适合什么基金」。
- 用户组合过于集中时，推荐「分散配置和行业集中度」。

## 四、用户流程

### 流程 A：新手自然语言选基

1. 用户进入首页，点击「AI智能选基」。
2. 默认进入新手模式。
3. 用户输入需求：「我想找一个波动小、适合长期定投的基金」。
4. AI 展示转译条件：中低风险、低回撤、适合定投、3 年以上业绩可参考。
5. 用户确认筛选。
6. 系统展示推荐结果列表。
7. 用户点击基金卡片，查看适配原因、核心风险和建议持有周期。
8. 用户可执行加入自选、对比、查看定投方案等动作。

### 流程 B：专业条件筛选

1. 用户切换到专业模式。
2. 选择基金类型、收益区间、最大回撤、基金经理年限、风险等级。
3. 可使用 AI 辅助输入复杂目标。
4. 系统展示结果，并保留解释层。
5. 用户进入详情页进行同类对比和风险查看。

### 流程 C：持仓收益拆解与风险预警

1. 用户进入「AI持仓管家」。
2. 查看今日收益、累计收益和 AI 持仓日报。
3. 查看收益拆解，理解主要贡献和拖累项。
4. 查看持仓健康度，包括集中度、波动、回撤、风格分布。
5. 点击风险预警中心。
6. 查看高 / 中 / 低风险预警及对应建议。
7. 选择继续观察、调整比例或学习相关知识。

### 流程 D：场景化投教成长

1. 用户进入「新手投教成长」。
2. 系统展示基于当前持仓和风险状态的推荐课程。
3. 用户完成一个短课程或互动问答。
4. 系统更新积分、进度和错题本。
5. 用户返回持仓或选基流程，形成「操作 - 学习 - 再操作」闭环。

## 五、信息架构

```text
首页
├─ AI智能选基
│  ├─ 新手模式
│  ├─ 专业模式
│  ├─ 推荐结果
│  └─ 基金详情 / 对比
├─ AI持仓管家
│  ├─ 持仓日报
│  ├─ 收益拆解
│  ├─ 持仓健康度
│  └─ 风险预警中心
└─ 新手投教成长
   ├─ 我的专属投教
   ├─ 场景课程
   ├─ 互动问答
   └─ 任务 / 积分 / 进度
```

## 六、页面优先级

### MVP 必做

- 首页 Dashboard
- AI智能选基页
- 选基结果页
- 基金详情页
- AI持仓管家页
- 风险预警页
- 新手投教成长页
- 课程 / 任务反馈页或弹层

### 可选增强

- 基金对比抽屉
- AI 条件转译动画
- 持仓健康度雷达图
- 投教错题本
- 面试演示模式入口

## 七、mock 数据策略

Demo 数据需要围绕同一位虚拟用户构建，保证三个模块之间有关联。

建议虚拟用户：

- 用户名：林然
- 投资阶段：基金新手，已有少量持仓
- 风险承受：中等偏低
- 当前目标：长期定投、降低波动、理解持仓收益

数据关系：

- 选基推荐结果中的基金，可以出现在持仓或自选中。
- 持仓风险预警应能触发投教推荐。
- 投教任务应能回流到持仓建议或选基解释。

## 八、设计边界

- 不提供真实交易。
- 不接入真实账户。
- 不展示真实实时行情。
- 不承诺收益。
- 所有 AI 结果均为本地 mock，用于说明产品体验。

## 九、成功标准

一个完整面试演示中，面试官应能看清：

- 这个项目解决了哪些用户问题。
- 三个模块之间不是孤立页面，而是一个「选前 - 买后 - 成长」闭环。
- AI 的作用是降低理解成本、提升解释质量和增强陪伴，而不只是包装概念。
- Demo 具备点击流程、数据内容和产品表达能力。
